# 项目优化方案

本文档基于当前代码结构审计，目标是把项目从“功能堆叠可运行”推进到“核心链路稳定、模块边界清晰、后续功能可持续迭代”。优先级按风险和收益排序。

## 1. 当前结论

项目已经具备简历编辑、模板切换、AI 生成、OCR、PDF/Word 导出、设计工作室、IndexedDB 持久化等能力，但主要问题是：核心页面过重、状态边界不清、导出方案重复、部分入口未闭环、遗留组件和测试页混在生产代码里。

高风险点：

- `src/views/HomeView.vue` 约 37KB，承担编辑器布局、模块导航、AI 推荐、模板管理、PDF/Word 导出、多页预览、移动端切换等职责，是当前最重的页面。
- `src/stores/resume.js` 约 35KB，同时管理简历数据、模板、全局样式、章节配置、IndexedDB/localStorage 迁移、自定义模块、自动保存，已经超过单一 store 的合理边界。
- `src/utils/ai/aiService.js` 约 35KB，统一 AI 接入后仍包含底层请求、简历业务 prompt、JSON 解析、职业模板和兼容旧 API，后续应拆分。
- PDF 导出存在多套方案：`browserPrint`、`printResume`、`pdfGenerator`、`multiPagePdfGenerator`、`templates/utils/pdfExport`，调用入口分散，行为不一致。
- 部分页面存在“按钮已展示但功能未实现”的情况，如 AI 助手页的 JD 上传、历史记录加载，设置页的云端存储、图片导出、水印、语言切换。
- 代码里存在运行时风险：`SettingsView.vue` 调用 `resumeStore.updateSettings`，`ResumeBuilderView.vue` 调用 `resumeStore.setExporting/isExporting`，当前 store 未提供这些 API。
- 存在疑似遗留组件：`src/components/resume/ResumeBuilder.vue` 引用了 `SectionOrderManager.vue` 和 `../muban/TemplateManager.vue`，这些路径当前不存在或已迁移。
- `setupAntiScrape` 在生产环境使用 debugger trap，可能伤害真实用户体验、浏览器兼容和可信度，不适合作为简历工具的默认策略。

## 2. 优化目标

第一阶段目标：

- 保证核心编辑、预览、保存、模板切换、导出、AI 配置不出现运行时错误。
- 删除或隔离遗留页面、测试页和未闭环入口。
- 统一数据保存、设置保存、导出和 AI 调用边界。

第二阶段目标：

- 拆分过大的页面和 store。
- 建立简历领域模型和模板渲染契约。
- 统一设计系统和页面布局规范。
- 为关键功能补测试和可观测错误提示。

第三阶段目标：

- 增加真正提升产品价值的功能：版本历史、简历评分、JD 匹配、ATS 检查、模板导入导出、云同步或本地备份。

## 3. 第一阶段：稳定性修复

优先修复会直接影响用户操作的运行时错误。

1. 补齐或移除不存在的 store API。

   当前 `SettingsView.vue` 调用：

   - `resumeStore.updateSettings`

   当前 `ResumeBuilderView.vue` 调用：

   - `resumeStore.isExporting`
   - `resumeStore.setExporting`

   推荐处理：

   - 如果 `ResumeBuilderView.vue` 不再作为路由入口，删除该页面或移动到 `src/legacy/`。
   - 如果仍保留，则在 store 中补齐 `isExporting` 和 `setExporting`。
   - `SettingsView.vue` 不应直接更新 resume store 的自动保存设置，应该把应用设置抽成独立 `settingsStore`。

2. 隐藏未实现功能入口。

   需要先隐藏或改为禁用态：

   - AI 助手页：上传 JD。
   - AI 助手页：历史记录加载。
   - 设置页：语言切换。
   - 设置页：云端存储。
   - 设置页：图片导出。
   - 设置页：水印。
   - 工程简历助手：ATS 优化、AI 技术总结。

   原则：没有完整数据流和错误处理的功能，不应该以可点击主入口出现。

3. 处理生产反调试逻辑。

   `src/utils/security.js` 中的 debugger trap 建议删除或改成仅内部演示环境启用。这个项目是简历工具，不是版权保护型内容站，强拦截开发者工具会造成性能和信任问题。

4. 清理遗留组件。

   优先检查并处理：

   - `src/components/resume/ResumeBuilder.vue`
   - `src/views/ResumeBuilderView.vue`
   - `src/views/Index.vue`
   - `src/views/RoutingTestView.vue`
   - `src/views/TestRandomStyleView.vue`
   - `src/components/layout/AppHeader.vue`

   推荐策略：

   - 路由未引用、业务未引用的组件移动到 `src/legacy/` 或删除。
   - 测试页面移动到 `tests/fixtures` 或仅在开发路由中注册。

5. 修复 HTML 注入风险。

   `AIAssistantView.vue` 和 `AdvancedAIResumeGenerator.vue` 使用 `v-html` 渲染 AI 内容。当前只是简单 replace markdown，缺少净化。

   推荐：

   - 引入统一 `renderMarkdownSafe(content)`。
   - 用 DOMPurify 或白名单渲染。
   - AI 返回内容默认视为不可信输入。

## 4. 第二阶段：架构重构

### 4.1 拆分 store

当前 `resumeStore` 职责过多，建议拆为：

- `resumeDataStore`：个人信息、简介、工作经历、教育、技能、项目、自定义模块数据。
- `templateStore`：当前模板、模板设置、模板分类和模板选择。
- `styleStore`：主题、字体、间距、布局、页面设置。
- `settingsStore`：自动保存、语言、AI 配置、导出偏好、缓存策略。
- `documentStore`：当前简历 ID、保存状态、更新时间、版本历史。

迁移方式：

1. 先新增 `settingsStore`，把 `SettingsView.vue` 的 localStorage 逻辑迁入。
2. 再新增 `styleStore`，把 `globalSettings` 拆出。
3. 最后把 IndexedDB 保存层改为 repository，store 不直接关心 Dexie 细节。

### 4.2 拆分 HomeView

`HomeView.vue` 应该只保留页面编排，不直接处理导出、AI 推荐、模板弹窗、移动端复杂状态。

建议拆分：

- `ResumeWorkspace.vue`：三栏/移动端布局容器。
- `ResumeSidebar.vue`：左侧模块、模板、AI 推荐入口。
- `ResumeEditorPane.vue`：中间表单编辑区。
- `ResumePreviewPane.vue`：预览、缩放、页码控制。
- `useResumeExport()`：PDF/Word 导出。
- `useResumeRecommendations()`：AI 推荐和完整度提示。
- `useMobileWorkspaceTabs()`：移动端 tab 状态。

拆分后 `HomeView.vue` 应控制在 120 行以内。

### 4.3 统一导出服务

当前导出工具分散，应建立一个统一入口：

```text
src/services/export/
├── exportService.js
├── pdfPrintExporter.js
├── pdfCanvasExporter.js
├── wordExporter.js
└── exportTypes.js
```

统一 API：

```js
exportResume({
  type: 'pdf-print' | 'pdf-canvas' | 'docx',
  elementId,
  resumeData,
  filename,
  pageSettings,
})
```

推荐默认策略：

- 默认 PDF：浏览器打印，保证文字清晰、文件可选文字、分页更符合打印语义。
- 高保真图片型 PDF：作为备选导出，不作为默认。
- Word：继续使用 `docx`，但加模板字段映射校验。

### 4.4 拆分 AI 服务

当前 AI 层应该拆成基础 provider 和业务 prompt。

建议结构：

```text
src/services/ai/
├── providers/
│   ├── openaiProvider.js
│   ├── geminiProvider.js
│   ├── ollamaProvider.js
│   └── localProvider.js
├── prompts/
│   ├── resumePrompts.js
│   ├── jdPrompts.js
│   └── ocrPrompts.js
├── aiService.js
└── aiResponseParser.js
```

拆分规则：

- provider 只负责请求和流式解析。
- prompt 只负责业务提示词。
- parser 只负责 JSON 修复和结构校验。
- 组件只调用业务函数，不拼 prompt。

### 4.5 统一模板渲染契约

当前存在大量 Vue 模板和 JSON Schema 模板，`templates/index.js` 注册表过长。

建议：

- 每个模板导出统一元信息：`id/name/category/features/suitableFor/component/isMultiPage/schema`。
- 模板注册从集中巨型对象改为自动发现 `template.meta.js`。
- 模板组件统一接收：

```js
{
  resumeData,
  templateId,
  pageSettings,
  sectionOrder,
  styleVars,
  renderMode: 'preview' | 'print' | 'thumbnail'
}
```

### 4.6 建立领域模型

当前字段存在不一致：

- 工作经历同时出现 `jobTitle`、`position`、`responsibilities`、`description`、`achievements`。
- 教育经历同时出现 `graduationDate`、`startDate`、`endDate`。
- 项目经历 `highlights` 和编辑器字段是否一致需要统一。

建议新增：

```text
src/domain/resumeSchema.js
src/domain/normalizers.js
src/domain/validators.js
```

所有 AI、OCR、导入、默认数据、模板渲染都必须先经过 normalizer。

## 5. 第三阶段：功能重新设计

### 5.1 AI 助手

当前 AI 助手是“聊天页”，但和简历编辑器联动弱。应改为“任务型 AI 面板”。

推荐任务：

- 生成个人简介。
- 优化选中工作经历。
- 根据 JD 提取关键词。
- 对当前简历做评分。
- 生成修改建议并允许一键应用。
- 对比修改前后差异。

需要补齐：

- 停止生成。
- 流式输出。
- 历史持久化。
- 错误重试。
- prompt 版本记录。
- AI 输出结构校验。

### 5.2 设置页

设置页应拆成：

- 应用设置。
- AI 设置。
- 导出设置。
- 数据与备份。
- 隐私与安全。

未实现功能应先隐藏，等数据流闭环后再开放。

### 5.3 导出

导出应提供明确模式：

- 快速打印 PDF。
- 高保真图片 PDF。
- Word 文档。
- JSON 备份。

不建议继续保留“图片导出”开关，除非实现 PNG/JPEG 导出。

### 5.4 模板中心

模板中心当前能选择模板，但筛选维度粗糙。

建议新增：

- 按职位、页数、ATS 友好、是否带照片、版式风格筛选。
- 模板收藏。
- 最近使用。
- 使用当前简历数据生成真实缩略图。
- 预览前后对比。

### 5.5 数据管理

建议新增：

- 手动备份 JSON。
- 从 JSON 恢复。
- 简历版本历史。
- 多份简历管理。
- 保存状态提示。
- IndexedDB 容量和失败提示。

## 6. 耦合严重区域

优先重构顺序：

1. `HomeView.vue`

   直接耦合：resume store、模板系统、PDF 导出、Word 导出、多页管理、AI 推荐、移动端状态、模板弹窗、OCR、打赏组件。

2. `resumeStore`

   直接耦合：IndexedDB、localStorage、简历数据、模板、样式、分页、章节配置、自定义模块、默认数据。

3. `aiService`

   直接耦合：provider、prompt、JSON parser、职业模板、简历业务函数、旧 Gemini 命名兼容。

4. 导出工具

   多个导出工具彼此功能重叠，页面直接选择具体实现，导致行为不可预测。

5. 模板系统

   注册表过长，模板元数据、组件映射、特殊命名规则集中在一个文件中。

## 7. 不合理或需要重新实现的功能

必须重做：

- AI 助手页历史记录：当前只创建内存历史，加载逻辑未实现。
- JD 上传：当前只是提示开发中，应实现文本粘贴、文件导入、JD 结构化和一键应用建议。
- 设置页云端存储：当前无后端或同步协议，不应展示为可选项。
- 图片导出、水印：当前设置有选项但导出链路未实现。
- 生产反调试：不符合工具型产品体验，建议移除。
- 旧 `ResumeBuilder.vue`：存在断裂引用，应删除或迁移。

建议降级：

- 关于页的“云端同步”“官方网站”“客服邮箱”等营销内容，当前与实际能力不一致，应改为真实说明。
- AI 状态在线检测当前恒为 true，应改成真实检测或隐藏。
- 打赏二维码组件不应默认悬浮在核心编辑体验中，建议移到关于页或设置页。

## 8. 建议新增功能

短期高价值：

- JSON 备份/恢复。
- 保存状态提示。
- AI 设置检测和模型列表读取。
- JD 粘贴分析。
- 当前简历完整度评分。
- 一键导出前检查：姓名、联系方式、空模块、页数溢出。

中期高价值：

- 多简历管理。
- 版本历史和回滚。
- 模板收藏和最近使用。
- ATS 检查。
- AI 修改 diff 预览。
- 本地模型加载进度页。

长期功能：

- 云同步。
- 团队模板库。
- 简历投递记录。
- 多语言简历。
- 职位库和关键词库。

## 9. 推荐实施顺序

### Sprint 1：止血

- 隐藏未实现入口。
- 删除或隔离遗留组件和测试页。
- 补齐缺失 store API 或移除调用。
- 统一 `v-html` 安全渲染。
- 删除生产 debugger trap。

### Sprint 2：拆边界

- 新增 `settingsStore`。
- 抽 `useResumeExport`。
- 抽 `useResumeRecommendations`。
- 把 `HomeView` 拆成 3 到 5 个业务组件。
- 把 AI provider 和 prompt 拆开。

### Sprint 3：统一契约

- 新增简历 schema 和 normalizer。
- 统一工作经历、教育、项目字段。
- 统一模板 props。
- 统一导出 API。

### Sprint 4：产品增强

- JD 分析闭环。
- 简历评分和修改建议。
- 备份/恢复。
- 多简历管理。
- 模板收藏和最近使用。

## 10. 验收标准

每轮优化至少满足：

- `npm run build` 通过。
- 首页编辑、保存、刷新恢复正常。
- 模板切换后预览正常。
- PDF 和 Word 至少各有一条主路径可用。
- 设置页不触发运行时错误。
- AI 配置失败时有清晰错误提示。
- 移动端编辑、预览切换不遮挡底部操作。
