# 功能建议清单

本文档只记录可以加入的功能方向，不代表全部都应该立即实现。建议优先选择能提升核心链路稳定性和用户完成简历效率的功能，避免继续堆叠低闭环入口。

## 1. 功能优先级结论

优先做：

- 简历质量检查和导出前检查。
- JD 匹配和 ATS 关键词分析。
- 版本历史和一键恢复。
- 本地备份、导入、导出 JSON。
- AI 润色入口标准化。
- PDF 导出稳定性和失败诊断。

第二阶段做：

- 多简历管理。
- 模板收藏和模板预设。
- 求职岗位库。
- 简历投递记录。
- Markdown/Word 导入解析。
- 面试题生成。

谨慎做：

- 云同步。
- 在线账号系统。
- 大量营销型模板商城。
- 复杂拖拽布局编辑器。
- 过多 AI Agent 自动流程。

## 2. 核心编辑功能

### 2.1 导出前检查

用户价值：避免导出空简历、联系方式缺失、头像异常、章节为空、内容溢出等问题。

建议检查项：

- 姓名为空。
- 电话、邮箱至少缺一个。
- 个人简介为空。
- 工作经历、项目经历、教育经历全部为空。
- 技能列表为空。
- 存在空标题、空公司、空学校、空项目名。
- 简历超过 1 页或 2 页时提醒。
- 头像加载失败或图片过大。

建议入口：

- 导出 PDF 前自动弹出检查结果。
- 首页预览区域显示一个简短状态，例如“3 个问题待处理”。
- 每个问题可以点击跳转到对应编辑模块。

实现要点：

- 新增 `useResumeValidation()`。
- 检查逻辑只依赖 resume store 数据，不依赖 DOM。
- 输出统一结构：`level`、`section`、`message`、`action`。

### 2.2 章节内容完整度

用户价值：让用户知道简历哪里还没写完，而不是只看到一个空白预览。

建议内容：

- 每个章节显示完成度。
- 空章节显示“建议补充”的轻提示。
- 对工作经历、项目经历、技能分别给出字段级缺失提示。

建议入口：

- 首页左侧章节列表。
- 导出前检查弹窗。
- AI 助手侧边栏。

### 2.3 多简历管理

用户价值：用户可以为不同岗位维护不同版本，而不是反复覆盖同一份简历。

建议功能：

- 新建简历。
- 复制当前简历。
- 重命名。
- 删除。
- 最近编辑时间。
- 按目标岗位筛选。

实现要点：

- IndexedDB 中用 `resumeId` 管理多份简历。
- 当前 store 只保存当前打开的简历。
- 切换简历时先保存当前草稿。

## 3. AI 功能

### 3.1 AI 润色标准入口

用户价值：用户在编辑任意文本时都能快速润色，而不是只能在单独 AI 页面操作。

建议入口：

- 个人简介：生成、润色、压缩到 100 字、改成更专业。
- 工作经历：改成 STAR 表达、量化成果、压缩每条职责。
- 项目经历：补充技术难点、项目成果、业务价值。
- 技能：按岗位重新分类、删除低相关技能。

实现要点：

- 每个编辑器不要直接写 prompt。
- 新增 `src/utils/ai/resumePrompts.js` 管理 prompt。
- AI 返回结果先展示预览，用户确认后再写入 store。

### 3.2 JD 匹配分析

用户价值：这是简历工具里最有价值的 AI 功能之一，可以直接帮助用户针对岗位优化。

建议功能：

- 粘贴 JD 文本。
- 分析匹配分数。
- 提取岗位关键词。
- 列出简历中已覆盖和未覆盖的关键词。
- 给出章节级修改建议。
- 一键生成针对该 JD 的个人简介和技能排序建议。

建议输出：

- 匹配度分数。
- 必须补充的关键词。
- 可选增强关键词。
- 简历风险点。
- 推荐修改的章节。

实现要点：

- 不要让 AI 直接覆盖整份简历。
- 先生成建议，再由用户选择应用到具体字段。
- 保存 JD 分析历史，方便不同岗位对比。

### 3.3 ATS 检查

用户价值：提升简历被招聘系统解析和筛选的概率。

建议检查项：

- 是否有联系方式。
- 是否使用过多图片或装饰。
- 是否存在不常见章节标题。
- 是否有过长段落。
- 是否缺少岗位关键词。
- 是否有表格、复杂布局导致解析风险。
- 日期格式是否统一。

建议入口：

- 导出前检查。
- AI 助手页。
- 首页预览顶部状态。

### 3.4 AI 修改记录

用户价值：用户能知道 AI 改了什么，避免被直接覆盖。

建议功能：

- 显示原文和建议文本对比。
- 支持接受、拒绝、复制。
- 支持回滚最近一次 AI 修改。

实现要点：

- 修改前保存字段快照。
- AI 操作统一走一个 apply 层，不让组件直接改 store。

## 4. 模板和预览

### 4.1 模板预设

用户价值：降低设计设置复杂度，让用户快速切换风格。

建议预设：

- 通用简洁。
- 技术工程。
- 产品运营。
- 校招应届。
- 高级管理。
- 学术科研。

每个预设包含：

- 模板 ID。
- 主色。
- 字体。
- 字号。
- 章节顺序。
- 是否显示头像。
- 头像位置。

### 4.2 模板收藏

用户价值：用户可以保留常用模板组合。

建议功能：

- 收藏当前模板和样式设置。
- 重命名收藏。
- 一键应用收藏。
- 删除收藏。

实现要点：

- 收藏的是配置，不是复制模板组件。
- 配置存入 localStorage 或 IndexedDB。

### 4.3 预览交互增强

用户价值：减少编辑器和预览之间的跳转成本。

建议功能：

- 点击预览中的章节，左侧自动切到对应编辑器。
- 预览显示当前页数。
- 内容溢出时标记溢出的章节。
- 支持预览缩放记忆。

实现要点：

- 模板渲染时给章节根节点加 `data-section`。
- 点击事件只做导航，不直接编辑 DOM。

## 5. 导出功能

### 5.1 PDF 失败诊断

用户价值：当 PDF 导出失败时，用户能知道应该怎么处理。

建议诊断项：

- 图片跨域或头像加载失败。
- DOM 宽高为 0。
- html2canvas 渲染异常。
- 页面样式包含不兼容 CSS。
- 内容过长导致内存不足。

建议提示：

- 显示具体失败原因。
- 提供“隐藏头像后重试”。
- 提供“使用浏览器打印导出”。

### 5.2 浏览器打印导出

用户价值：当 canvas PDF 方案失败时，有一个更稳定的备选路径。

建议功能：

- 独立打印页。
- 使用 `window.print()`。
- 保留可选中文本。
- 打印 CSS 控制分页。

实现要点：

- 打印页只渲染简历，不带编辑器 UI。
- 打印样式和预览样式共用模板变量。

### 5.3 Word 和 Markdown 导出

用户价值：满足用户后续手工修改、投递平台粘贴、备份等需求。

建议功能：

- Markdown 导出。
- 纯文本导出。
- Word 导出使用结构化数据，不依赖截图。

## 6. 数据安全和备份

### 6.1 一键备份和恢复

用户价值：避免浏览器缓存清空后简历丢失。

建议功能：

- 导出当前简历 JSON。
- 导出全部简历 JSON。
- 导入 JSON。
- 导入前预览简历名称和更新时间。

实现要点：

- 增加数据版本号 `schemaVersion`。
- 导入时做字段迁移和校验。

### 6.2 自动版本历史

用户价值：用户可以恢复误删内容。

建议功能：

- 每次重要修改保存快照。
- 每 5 分钟最多保存一个自动版本。
- 保留最近 20 个版本。
- 支持查看差异和恢复。

实现要点：

- 不保存头像原始大图多份，避免 IndexedDB 过大。
- 快照按 resumeId 分组。

### 6.3 隐私模式

用户价值：简历包含敏感个人信息，需要让用户明确数据存放位置。

建议功能：

- 设置页展示“数据仅保存在本机浏览器”。
- 提供清空全部本地数据。
- AI 调用前提示会把相关文本发送到第三方接口。
- 支持关闭 AI 功能。

## 7. 求职流程功能

### 7.1 投递记录

用户价值：把简历制作和求职管理连接起来。

建议字段：

- 公司。
- 岗位。
- JD 链接。
- 使用的简历版本。
- 投递日期。
- 当前状态。
- 备注。

建议状态：

- 待投递。
- 已投递。
- 笔试。
- 面试。
- Offer。
- 拒绝。

### 7.2 面试准备

用户价值：基于简历内容生成面试准备材料。

建议功能：

- 根据项目经历生成面试问题。
- 根据 JD 生成可能追问。
- 生成自我介绍。
- 生成项目讲解稿。

实现要点：

- AI 输入应包含用户选择的项目，而不是整份简历全部发送。
- 输出可保存到本地笔记。

## 8. 设置和可用性

### 8.1 AI 配置诊断

用户价值：减少“连接失败但不知道为什么”的问题。

建议诊断项：

- 当前引擎。
- Base URL。
- 模型名。
- API Key 是否为空。
- 请求状态码。
- 服务端错误信息。
- 是否可能被 CORS 阻止。

建议入口：

- 设置页连接测试下方展示诊断详情。

### 8.2 新手默认数据

用户价值：打开应用后不再是一堆空字段，用户能直接理解效果。

建议功能：

- 提供示例简历。
- 一键清空示例。
- 选择目标岗位后生成基础章节结构。

### 8.3 快捷操作

用户价值：提升频繁编辑效率。

建议功能：

- 复制章节。
- 折叠章节。
- 快速新增工作经历、项目经历、技能。
- 批量删除空条目。
- 拖拽排序条目。

## 9. 暂不建议加入的功能

### 9.1 复杂自由布局编辑器

原因：

- 会显著增加模板、预览、PDF 导出的复杂度。
- 容易导致 ATS 不友好。
- 当前用户已经明确不需要设计空间布局功能。

建议：

- 保留模板、章节顺序、头像位置、字体、颜色、间距这些可控配置即可。

### 9.2 账号和云同步

原因：

- 需要后端、登录、数据安全、隐私合规。
- 简历数据敏感，不能草率上线。

建议：

- 先做好本地备份和恢复。
- 后续如果做云同步，必须先设计加密、导出、删除账号、数据迁移方案。

### 9.3 模板商城

原因：

- 会让产品重心偏向视觉数量，而不是简历完成质量。
- 维护成本高。

建议：

- 先做好少量高质量模板和模板预设。

## 10. 推荐实施顺序

第一批：稳定核心体验。

1. 导出前检查。
2. PDF 失败诊断。
3. 一键 JSON 备份和恢复。
4. AI 配置诊断详情。
5. 章节完整度提示。

第二批：提升简历质量。

1. JD 匹配分析。
2. ATS 检查。
3. AI 字段级润色。
4. AI 修改前后对比。
5. 模板预设。

第三批：扩展求职流程。

1. 多简历管理。
2. 版本历史。
3. 投递记录。
4. 面试准备。
5. 求职岗位库。

## 11. 最小可行版本建议

如果只选 5 个功能，建议做：

1. 导出前检查。
2. JD 匹配分析。
3. ATS 检查。
4. 版本历史。
5. JSON 备份和恢复。

这 5 个功能对简历工具最有实际价值，能直接解决用户“写得好不好、能不能投、会不会丢、能不能导出”的核心问题。


可以加，但别堆“花活”。这个页面现在已有：配色、字体、字号、行高、页数、页边距、重置。下一步最值得做的是让用户能控制“简历视觉结构”，而不是只调颜色字号。

  我建议按优先级加这些：

  1. 模板版式风格
  加一个“版式” Tab，但保持单栏方向：

  - 经典单栏：标题下划线，适合 ATS
  - 现代单栏：左侧强调线、模块分隔线
  - 商务页眉：顶部深色姓名栏
  - 极简文本：去掉大色块，只保留字号和间距
  - 重点突出：项目、技能模块更醒目

  这个最直接解决“模板难看”的问题。用户不是想调一个颜色，而是想换整体感觉。

  2. 模块标题样式
  现在模块标题样式比较固定，可以让用户选：

  - 无装饰标题
  - 下划线标题
  - 左竖线标题
  - 背景条标题
  - 大写英文/中文标题风格
  - 标题居左/居中

  这比加一堆模板更轻量，也能显著改变观感。

  3. 信息密度控制
  加“舒展 / 标准 / 紧凑”三档：

  - 模块间距
  - 条目间距
  - 行高
  - 技能标签间距
  - 工作经历 bullet 间距

  现在只有单独滑杆，用户不一定知道怎么调。三档预设会更好用。

  4. 头像区域设计
  如果简历有头像，可以加：
  - 头像大小
  - 头像位置：左 / 中 / 右
  - 是否显示头像边框
  - 头像与姓名同行 / 上下排列

  你现在已有头像位置逻辑，可以继续扩展。

  5. 联系方式样式
  联系方式很影响简历第一眼：

  - 横向一行
  - 自动换行
  - 图标 + 文本
  - 纯文本分隔符
  - 紧凑小字
  - 是否显示分隔点 / | ·

  注意：图标要克制，简历里不要太花。

  6. 技能展示样式
  技能模块可以单独设计：

  - 标签样式
  - 纯文本逗号分隔
  - 分组列表
  - 熟练度显示 / 隐藏
  - 技能标签边框 / 背景 / 无底色

  这对技术岗、设计岗很有用。

  7. 项目/经历时间样式
  经历条目的时间现在在右侧，可以加：

  - 时间右对齐
  - 时间放标题下方
  - 时间用浅色标签
  - 公司和职位谁更突出
  - 是否显示地点

  这类控制很实用，尤其不同岗位简历重点不同。

  8. 一键风格预设
  保留当前细调能力，但加“组合预设”：

  - 大厂技术
  - 产品运营
  - 应届校招
  - 管理商务
  - 设计创意
  - 外企 ATS
  - 国企简洁

  这些预设本质上是一次性写入 theme、typography、spacing、layout，不复杂，但用户感知很强。

  9. 设计质量检查
  加一个“检查”区，比单纯设计更有价值：

  - 字号是否过小
  - 一页是否过满
  - 对比度是否不足
  - 模块间距是否过密
  - 联系方式是否太长
  - 是否存在空模块
  - 是否适合 ATS

  这个适合放在设计页右上角，点击后给出问题列表。

  10. 撤销 / 恢复 / 保存方案
  设计页非常需要：

  - 撤销上一步
  - 恢复
  - 保存当前样式为“我的样式”
  - 复制当前样式
  否则用户调坏了会没有安全感。

  我建议下一步先做这三个，收益最高：

  1. 新增 版式风格 Tab
  2. 新增 模块标题样式 控制
  3. 新增 信息密度：舒展 / 标准 / 紧凑

  这三项能明显改变页面质感，而且和现有 globalSettings 结构最容易接上。
