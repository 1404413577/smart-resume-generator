# 简历生成器 — 项目优化计划

> 更新时间：2026-03-05

---

## 一、体验与 UI 优化

### 1.1 模板中心
- [ ] **网格卡片高度统一**：当前 `TemplatePreview` 按 A4 比例自动计算高度，网格行高不一致，需加 `max-height` 裁剪并统一卡片尺寸
- [ ] **懒加载模板预览**：16 个模板同时渲染 Vue 组件会造成较大开销，改为 `IntersectionObserver` 滚动到可视区才渲染
- [ ] **模板收藏/标星**：用 localStorage 存储用户收藏，方便快速定位常用模板
- [ ] **推荐模板分类算法**：目前只按 category 筛选前 3 个，后续可根据用户使用记录智能推荐

### 1.2 编辑器体验
- [ ] **自动保存提示**：显示"上次保存时间"，引导用户确认数据已持久化
- [ ] **撤销/重做（Undo/Redo）**：对简历内容的所有修改支持 Ctrl+Z / Ctrl+Y 操作
- [ ] **拖拽排序章节**：章节排序目前通过按钮逐步移动，改为支持拖拽（`sortable.js` 或 Vue Draggable）
- [ ] **富文本编辑器**：工作经历/项目描述支持加粗、斜体、列表等基础格式（引入 Quill 或 Tiptap）

### 1.3 预览区
- [ ] **实时缩放手势**：移动端支持双指捏合缩放预览
- [ ] **预览区固定高度可滚动**：目前有溢出裁剪问题，需确保长简历可以滚动浏览
- [ ] **分页线显示**：在预览区画出 A4 分页虚线，让用户直观看到哪里会换页

---

## 二、样式设置功能

### 2.1 CSS 变量覆盖面
- [ ] **旧模板适配**：`ClassicTemplate`、`AcademicTemplate`、`TechTemplate` 等早期模板仍使用硬编码颜色，需统一迁移到 `var(--resume-primary-color)` 等变量
- [ ] **字体预览**：字体选择器下拉菜单内每个选项用对应字体渲染，方便直观对比
- [ ] **主题预设扩展**：现有 3 个预设（professional / creative / minimal），补充若干行业主题（金融深蓝、教育暖橙、科技深灰等）

### 2.2 实时预览联动
- [ ] **颜色选择器防抖**：拖动颜色滑块时频繁触发重绘，加 `debounce(200ms)` 减少 CPU 压力
- [ ] **字体大小滑块数值显示**：滑块旁显示当前 px 值，方便精确设置

---

## 三、PDF 导出优化

### 3.1 打印模式（Plan B）
- [ ] **导出前预检**：点击导出前自动检查是否有内容为空的必填项，给出提示
- [ ] **打印样式覆盖完整性测试**：逐一验证所有 16 个模板在 @media print 下是否正常渲染（特别是 StartupTemplate 的深色背景是否压缩为白底）
- [ ] **深色/背景图片打印兼容**：`StartupTemplate` header 使用深色背景，Chrome 默认打印省略背景色，需在 CSS 中加 `-webkit-print-color-adjust: exact`

### 3.2 原 html2canvas 方案
- [ ] **图片清晰度**：目前 `scale: 3`，可提供用户选择"标准/高清"两档，高清用 `scale: 4`
- [ ] **中文字体嵌入**：导出 PDF 后在某些设备上中文显示为非嵌入字体，考虑使用 `jsPDF-Font` 或服务端渲染方案

## 四、性能优化

### 4.1 首屏加载
- [ ] **路由懒加载**：所有视图组件改为异步 `() => import(...)` 避免打包到主 chunk
- [ ] **模板组件懒加载**：`templateRegistry` 中所有 import 改为动态 import，按需加载
- [ ] **图片资源优化**：模板预览图（`/images/templates/*.png`）目前为占位路径，实际生成后应压缩到 < 50KB / 张

### 4.2 运行时性能
- [ ] **TemplatePreview 虚拟滚动**：16 个实时渲染的 Vue 模板组件同时存在 DOM 中开销大，可用虚拟列表或 `v-memo` 优化
- [ ] **减少 watcher 深度**：Pinia store 中对 `globalSettings` 的 `{ deep: true }` 监听在每次输入时都会重新计算，可拆分为多个浅层 watcher
- [ ] **防抖输入保存**：每次编辑都触发 `saveToLocalStorage`，改为 `debounce(1500ms)`

---

## 五、可靠性与代码质量

### 5.1 错误处理
- [ ] **模板加载失败降级**：模板组件加载异常时显示空状态而不是崩溃
- [ ] **localStorage 超限处理**：头像 base64 数据大可能超出 5MB 的 localStorage 上限，需 try/catch 并提示

### 5.2 代码重构
- [ ] **CSS 变量命名统一**：当前存在 `--theme-primary`、`--primary-color`、`--resume-primary-color` 三套命名并存，统一到一套规范
- [ ] **工具函数拆分**：`styleUtils.js` 已超过 300 行，将颜色工具、字体工具、打印工具各拆为独立文件
- [ ] **组件 Props 类型完善**：多个组件缺少 TypeScript/PropTypes 校验，加入后有助于 IDE 提示和错误定位

### 5.3 测试
- [ ] **单元测试**：为 `styleUtils.js` 的颜色合并逻辑、CSS 变量生成逻辑添加 Vitest 单元测试
- [ ] **E2E 测试**：用 Playwright 测试"填写信息 → 选模板 → 导出 PDF"的核心流程
